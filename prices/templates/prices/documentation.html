Documentation

How PerfumeX Prices currently works.

Products Page

  • Default filter: Status = Active.
  • Default currency view: USD.
  • Search matches by product name only (not SKU).
  • When opening a product and going back, search/filter state is kept.
  • Last opened product row/card is softly highlighted.

Manual Import (Supplier Import)

  • File types: .xlsx, .xls, .csv (if parser supports supplier format).
  • Mapping uses column indexes (1-based in UI).
  • Name can be mapped from multiple columns (comma-separated) and gets concatenated with spaces.
  • Optional currency column can be provided per row.
  • Rows are skipped when price is empty/invalid/zero or when name is clearly invalid.
  • Products missing from the latest import are marked inactive (not deleted).

Email Import

  • Import can run per supplier or for all suppliers.
  • System checks mailbox and applies supplier matching rules.
  • Duplicate detection is hash-based for attachments.
  • Status/progress is shown in Import Prices page.
  • You can cancel a running supplier email import.

History Rules

  • Price history is stored in snapshots with import timestamp.
  • For charts/tables, latest snapshot per day is used as daily history point.
  • If same day has multiple files, newer snapshots update that day’s latest state.
  • Product page “Last imported at” shows last successful product update timestamp.

Currencies

  • Rates are managed in Settings → Currencies.
  • CBR autofill can create/update rates using CBR value plus markup percent.
  • Products list conversion uses the latest available rate for display.
  • Historical snapshots can keep RUB and USD values based on historical date rate logic.

Backfill Types

  • Email backfill: imports historical supplier files from a date range.
  • Mixed currency backfill: one-time helper for ranges where older files are USD and newer are RUB.
  • Use “replace existing snapshots” carefully; it rewrites existing range data.

Known Practical Notes

  • After template/JS/CSS updates, use hard refresh (Ctrl+F5).
  • If new DB fields are added and page fails with no such column/table, run migrations.
  • For server deploy, GitHub Actions + SSH workflow applies code, migrations, static, and service restart.
