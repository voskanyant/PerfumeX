Reference

Technical documentation

Operational notes for developers, operators, and automation flows inside PerfumeX Prices.

System overview

  • Framework: Django with server-rendered templates and JSON endpoints for live search.
  • App process: Gunicorn via perfumex.service.
  • Web server: Nginx reverse proxy.
  • Database: PostgreSQL in production, SQLite optionally used locally.
  • Email ingest: IMAP mailboxes with attachment parsing.
  • Scheduler: cron calling python manage.py import_emails.

Core model

  • Supplier, Mailbox, SupplierMailboxRule
  • ImportBatch, ImportFile, EmailImportRun
  • SupplierProduct, PriceSnapshot, ExchangeRate
  • ImportSettings, UserPreference

Import pipeline

Manual upload

  • Upload creates an ImportBatch and ImportFile.
  • Supplier mapping determines how the file is parsed.
  • Successful processing writes current supplier prices and history snapshots.

Email import

  • Recovers stale pending rows from interrupted runs.
  • Applies interval rules unless forced.
  • Synchronizes daily CBR rates when needed.
  • Runs mailbox search and attachment dedupe by hash.
