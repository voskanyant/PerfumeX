Technical Documentation

Current project notes for developers, operators, and AI agents.

This page is a live summary of the current codebase. The detailed repo handoff lives in README.md and PROJECT_HANDOFF.md at the repository root.

  • README.md: setup, routes, stack, operations summary.
  • PROJECT_HANDOFF.md: detailed architecture, data flow, commands, constraints, and maintenance notes.

1) Current System Facts

  • Framework: Django 5, server-rendered templates, one main app: prices.
  • Database: PostgreSQL only. Current settings do not support SQLite runtime.
  • Static files: WhiteNoise.
  • Timezone: request rendering is forced to Europe/Moscow.
  • Viewer routes live on / and still require login.
  • Custom staff workspace lives under /admin/.
  • Django admin lives separately under /django-admin/.
  • Frontend policy: new UI work must use native PerfumeX layout and component classes. Do not introduce Bootstrap component styling or Bootstrap-style utility class patterns in new templates.
  • Approved Supplier Products style direction: keep the current refined layout language. Product names should stay readable and slightly prominent, desktop supplier labels keep the full pill treatment, mobile supplier labels stay plain bolder text without pill backgrounds, and helper controls like the detail icon should remain small and visually quiet.

2) Core Data Model

  • Supplier: supplier identity, defaults, email patterns, diagnostics.
  • Mailbox: IMAP settings and incremental UID cursors.
  • SupplierMailboxRule: mailbox-specific supplier matching.
  • SupplierFileMapping: spreadsheet parsing configuration.
  • ImportBatch: one import event for one supplier.
  • ImportFile: one imported file plus blob/hash/status/error.
  • SupplierProduct: current supplier catalog row.
  • PriceSnapshot: historical price record.
  • OurProduct: internal normalized product target.
  • ExchangeRate: dated conversion table.
  • ImportSettings: singleton operational settings.
  • EmailImportRun: import progress and detailed log.
  • UserPreference: per-user saved filters and exclude terms.

3) Import Pipeline

  1. Create ImportBatch and ImportFile.
  2. Find active supplier mapping.
  3. Parse spreadsheet rows from .csv, .xlsx, or .xls.
  4. Normalize price values and infer currency when needed.
  5. Build supplier identity keys from SKU or normalized name.
  6. Upsert SupplierProduct current-state rows.
  7. Create PriceSnapshot history rows.
  8. When the imported batch is the latest for that supplier, deactivate missing products.

The single most important backend entry point is prices.services.importer.process_import_file().

4) Email Import Mechanics

  • Main command: python manage.py import_emails.
  • Importer uses IMAP and can merge Gmail INBOX with All Mail.
  • Supplier matching priority: mailbox rule first, supplier fallback second.
  • Duplicate detection uses supplier + local-day window + SHA-256 content hash.
  • Interrupted stale pending imports are auto-failed and may be retried from stored files.
  • Daily CBR USD/RUB rate is synced automatically when needed.

5) Operations And Scheduler

  • UI path: Settings -> Import Settings.
  • Scheduler install/remove is handled from the web UI.
  • Run now starts an in-process background thread from the web app and calls import_emails --force.
  • Production deploy is defined in .github/workflows/deploy.yml.
  • Deploy target assumes app root /opt/perfumex/PerfumeX and service name perfumex.

6) Main UI Areas

  • Suppliers: supplier records, mappings, mailbox rules, manual import.
  • Supplier Overview: import operations dashboard and batch log.
  • Supplier Products: current supplier catalog with live search and deltas.
  • Product Detail: latest-per-day history chart and batch traceability.
  • Our Products: internal grouping target for linked offers.
  • Product Linking: suggestion scoring between supplier and internal products.
  • Currencies: exchange-rate maintenance.
  • Users / Groups / Mailboxes: staff operations and permissions.

7) Management Commands

  • import_emails: scheduled/manual mailbox import entry point.
  • cleanup_duplicate_price_imports: dedupe historical imports by supplier/day/hash.
  • import_supplier_folder: bulk import price files from a server folder.
  • reorganize_import_files: move stored files to organized paths.
  • repair_supplier_price_imports: reprocess stored processed price files.

8) Practical Caveats

  • The repository contains db.sqlite3, but runtime settings still require PostgreSQL.
  • Local helper run_python_server.cmd uses --noreload, so edits require restart.
  • Automated tests are effectively missing right now.
  • Import, dedupe, and historical conversion changes are high-risk and should be treated carefully.

9) Fast Debug Checklist

  1. Run python manage.py check.
  2. Confirm PostgreSQL environment variables are loaded.
  3. Confirm supplier mapping exists and matches the real file.
  4. Check latest EmailImportRun and import batch details.
  5. Check sender/subject/filename matching patterns.
  6. Check blacklist terms and duplicate hash behavior.
  7. If local UI changes look missing, restart the dev server because --noreload is in use.
